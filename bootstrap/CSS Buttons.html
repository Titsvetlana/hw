Передача смысла вспомогательным технологиям

Использование цвета для добавления значения обеспечивает только визуальную индикацию, которая не будет передаваться пользователям вспомогательных технологий, таких как программы чтения с экрана. Убедитесь, что информация, обозначенная цветом, либо очевидна из самого содержимого (например, видимого текста), либо включена альтернативными способами, такими как дополнительный текст, скрытый с .sr-only классом.

Теги кнопок

Эти .btn классы предназначены для использования с <button> элементом. Тем не менее, вы также можете использовать эти классы <a> или <input> элементы (хотя некоторые браузеры могут применять немного другой рендеринг).

При использовании классов кнопок для <a>элементов, которые используются для запуска внутристраничной функциональности (например, свертывание содержимого), вместо ссылки на новые страницы или разделы на текущей странице, эти ссылки должны быть предоставлены, role="button"чтобы надлежащим образом передать их назначение вспомогательным технологиям, таким как программы чтения с экрана.

Link 
			<a class="btn btn-primary" href="#" role="button">Link</a>
			<button class="btn btn-primary" type="submit">Button</button>
			<input class="btn btn-primary" type="button" value="Input">
			<input class="btn btn-primary" type="submit" value="Submit">
			<input class="btn btn-primary" type="reset" value="Reset">

Активное состояние

Кнопки будут отображаться нажатыми (с более темным фоном, более темной рамкой и вставленной тенью), когда они активны. Нет необходимости добавлять класс в <button>s, так как они используют псевдокласс . Тем не менее, вы все равно можете принудительно активировать тот же активный внешний вид .active(и включить aria-pressed="true"атрибут), если вам потребуется программно реплицировать состояние.

Primary link Link 
		<a href="#" class="btn btn-primary btn-lg active" role="button" aria-pressed="true">Primary link</a>
                <a href="#" class="btn btn-secondary btn-lg active" role="button" aria-pressed="true">Link</a>
Неработающее состояние

Сделайте кнопки неактивными, добавив disabled логический атрибут к любому <button> элементу.

Отключенные кнопки, использующие элемент, ведут себя немного иначе:

  1. <a>disabled Атрибут не поддерживается , поэтому вы должны добавить .disabled класс, чтобы визуально он был отключен.
  2. Некоторые перспективные стили включены, чтобы отключить все pointer-events кнопки привязки. В браузерах, которые поддерживают это свойство, вы вообще не увидите отключенный курсор.
  3. Отключенные кнопки должны включать aria-disabled="true" атрибут, указывающий состояние элемента для вспомогательных технологий.
Primary link Link 
	     <a href="#" class="btn btn-primary btn-lg disabled" role="button" aria-disabled="true">Primary link</a>
             <a href="#" class="btn btn-secondary btn-lg disabled" role="button" aria-disabled="true">Link</a>
Ссылка на функциональность

.disabled Класс использует , pointer-events: none чтобы попытаться отключить функциональность линии связи <a> с, но это свойство CSS еще не стандартизированы. Кроме того, даже в браузерах, которые поддерживают эту функцию pointer-events: none, навигация с помощью клавиатуры остается неизменной, а это означает, что зрячие пользователи клавиатуры и пользователи вспомогательных технологий по-прежнему смогут активировать эти ссылки. Поэтому для безопасности добавьте tabindex="-1" атрибут в эти ссылки (чтобы они не фокусировались на клавиатуре) и используйте пользовательский JavaScript, чтобы отключить их функциональность.

Плагин кнопки

Делай больше с кнопками. Управляйте состояниями кнопок или создавайте группы кнопок для дополнительных компонентов, таких как панели инструментов.

Переключить состояния

Добавить data-toggle="button" для переключения состояния кнопки active. Если вы предварительно переключаете кнопку, вы должны вручную добавить .active класс и aria-pressed="true" к <button>.

Переключить состояния

Добавить data-toggle="button" для переключения состояния кнопки active. Если вы предварительно переключаете кнопку, вы должны вручную добавить .active класс и aria-pressed="true" к <button>. 

<button type="button" class="btn btn-primary" data-toggle="button" aria-pressed="false" autocomplete="off"> Single toggle</button>
Флажок и радио кнопки

.button Стили Bootstrap могут применяться к другим элементам, таким как <label>s, чтобы обеспечить переключение переключателей или переключателей в стиле радио. Добавьте data-toggle="buttons" к .btn-group содержащим эти измененные кнопки, чтобы включить их переключение с помощью JavaScript, и добавьте .btn-group-toggle стили <input> в свои кнопки. Обратите внимание, что вы можете создавать отдельные кнопки с питанием от входа или их группы.

Проверенное состояние для этих кнопок обновляется только через click событие на кнопке. Если вы используете другой метод для обновления ввода-например, с <input type="reset">или вручную нанесением на вход в checked свойства-вы будете необходимости переключения .activeна <label>вручную.

Обратите внимание, что предварительно проверенные кнопки требуют, чтобы вы вручную добавили .active класс к входам <label>. 

		<div class="btn-group-toggle" data-toggle="buttons">
		  <label class="btn btn-secondary active">
		    <input type="checkbox" checked autocomplete="off"> Checked
		  </label>
		</div>
		
		<div class="btn-group btn-group-toggle" data-toggle="buttons">
		  <label class="btn btn-secondary active">
		    <input type="radio" name="options" id="option1" autocomplete="off" checked> Active
		  </label>
		  <label class="btn btn-secondary">
		    <input type="radio" name="options" id="option2" autocomplete="off"> Radio
		  </label>
		  <label class="btn btn-secondary">
		    <input type="radio" name="options" id="option3" autocomplete="off"> Radio
		  </label>
		</div>
